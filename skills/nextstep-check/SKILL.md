# nextstep-check — 넥스트스텝 미션 조건 체크

사용자가 붙여넣은 **미션 조건**을 현재 작업의 **diff**와 대조해, 조건별로 충족했는지 체크리스트로 피드백한다.
테스트/빌드는 실행하지 않고 **코드를 읽어 정적으로 대조**만 한다. 커밋 전·작업 중간에도 수시로 쓸 수 있다.

> 확인 대상은 `nextstep-pr`과 동일하게 **base 브랜치(본인 아이디) 대비 diff**다. 이번 단계에서 바꾼 코드에만 집중해 피드백한다.

## 1단계 — 컨텍스트 파악 (먼저 실행)

```bash
git remote -v               # origin URL의 github.com/{아이디}/{repo}에서 본인_아이디 추출 → base 브랜치 이름
git branch --show-current   # 작업 브랜치 (예: step2)
git diff {본인_아이디}...HEAD    # base 대비 커밋된 변경
git diff                    # working tree의 커밋 안 된 변경
git diff --cached           # staged 상태의 커밋 안 된 변경
```

- base 브랜치는 `origin` URL에서 추출한 **본인 아이디 브랜치**다. 로컬에 없으면 `origin/{본인_아이디}`를 쓴다.
- **커밋 안 된 변경도 확인 대상에 포함한다.** base 대비 커밋된 diff가 없고 working tree 변경만 있어도 그대로 진행한다.
- diff에 잡힌 파일이라도 변경 줄만으로 판단이 어려우면 해당 파일 전체를 읽어 문맥을 확인한다.
- base 대비 diff도, 커밋 안 된 변경도 전혀 없으면 "확인할 변경이 없다"고 알리고 종료한다.

## 2단계 — 조건 입력 확인

- 사용자가 대화에 붙여넣은 미션 조건(텍스트 또는 이미지)을 **항목 단위로** 정리한다.
- 조건 문구는 **원문 그대로** 옮긴다. 임의로 요약·왜곡·보충하지 않는다. (이미지면 보이는 문구를 그대로 옮겨 적는다)
- 조건이 아직 없으면 사용자에게 미션 조건을 붙여넣어 달라고 요청하고 기다린다.

## 3단계 — 대조

각 조건 항목을 diff(+ 필요 시 파일 문맥)와 1:1로 대조해 셋 중 하나로 분류한다.

| 상태 | 기준 |
|------|------|
| ✅ 충족 | 조건을 만족하는 코드를 diff에서 직접 확인함 |
| ❌ 미충족 | 조건에 해당하는 구현이 없거나, 조건과 다르게 동작하는 코드를 확인함 |
| ⏸ 판단보류 | 조건 문구가 모호하거나, 코드만 읽어서는(실행 없이) 충족 여부를 확정할 수 없음 |

- **애매하면 미충족이 아니라 판단보류다.** 억지로 이분하지 말고, 왜 판단이 어려운지 이유를 반드시 함께 적는다.
- 모든 상태에 근거를 붙인다: 충족/미충족은 해당 코드 위치(`파일:라인`)와 이유, 판단보류는 보류 사유.

## 4단계 — 출력

조건별 체크리스트로 정리해 **대화에만** 출력한다. 파일로 저장하지 않는다.

출력 형식 예:

```markdown
## 미션 조건 체크 결과 (step2, base: {본인_아이디})

- ✅ 자동차는 이름을 가진다 — `src/main/java/racing/Car.java:12` 생성자에서 name을 받아 보관
- ❌ 이름은 5자를 초과할 수 없다 — 이름 길이를 검증하는 코드가 diff에 없음
- ⏸ 우승자가 여러 명일 수 있다 — 조건은 확인되나 무작위 값에 의존하는 로직이라 정적 대조만으로 동작 확정 불가. `RacingGame.java:34`의 winners 계산은 복수 우승자를 허용하는 구조

**요약: 충족 1 · 미충족 1 · 판단보류 1**
```

- 마지막에 충족/미충족/판단보류 개수를 한 줄로 요약한다.
- 미충족 항목이 있으면 어디를 어떻게 고치면 되는지 간단히 덧붙여도 좋다(수정은 하지 않고 제안만).

## 하지 말 것

- 넥스트스텝 공통 코드 컨벤션(else 금지, 들여쓰기 depth 1, 메서드당 10줄 등)은 검사하지 않는다. **붙여넣은 미션 조건만** 본다.
- 테스트·빌드를 실행하지 않는다. 정적 대조만 한다.
- 검토 결과를 파일로 남기지 않는다. (대화 출력이 전부)
- 코드를 대신 수정하지 않는다. (피드백까지가 이 스킬의 책임)
- `nextstep-pr`·`nextstep-advance`를 자동으로 이어서 호출하지 않는다. (완전 독립 스킬)
- 사용자가 준 조건 문구를 임의로 요약·재해석해 원문과 다른 기준으로 판정하지 않는다.
